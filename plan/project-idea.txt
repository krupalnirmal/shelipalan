=====================================================
   शेळीपालन डिरेक्टरी / मार्केटप्लेस - प्रोजेक्ट आयडिया
=====================================================

मोटिव्ह (का बनवायचं?)
---------------------
गावात शेळीपालन मोठ्या प्रमाणात होतं. खातिक (बोकड/शेळी खरेदी करणारा दलाल)
दर महिन्याला ३-४ वेळा गावात येऊन घरोघरी विचारत फिरतो - "बोकड आहे का?"
यात त्याचा वेळ जातो आणि शेतकऱ्याचाही वेळ जातो (तो आला तेव्हा घरी
नसेल तर परत फेरी मारावी लागते).

उपाय: एक साधी वेबसाईट जिथे शेतकरी स्वतःच्या शेळ्या/बोकड/नवीन पिल्लांची
एंट्री टाकेल, आणि खातिक थेट वेबसाईटवर बघून ठरवेल कुणाकडे जायचं, काय
आहे, किती आहे, कुठे आहे. यामुळे दोघांचाही वेळ वाचेल आणि गावातल्या सगळ्या
शेळीपालकांची माहिती एकाच ठिकाणी मिळेल.


कोण वापरणार (Users)
--------------------
1. शेतकरी / शेळीपालक (Farmer) - एंट्री टाकणारा, स्वतःचा स्टॉक मॅनेज करणारा
2. खातिक / खरेदीदार (Buyer/Trader) - लिस्टिंग बघणारा, संपर्क करणारा
3. अॅडमिन (तुम्ही) - सगळं नियंत्रण, गावांची यादी, युजर्स वेरिफाय करणे


मुख्य फीचर्स (Core Features)
-----------------------------
1. शेतकरी रजिस्ट्रेशन / लॉगिन
   - नाव, मोबाईल नंबर + साधा पासवर्ड (कुठलाही OTP/SMS नाही - डायरेक्ट
     लॉगिन, गावात कुणालाही सहज वापरता यावं म्हणून सगळ्यात साधी पद्धत)
   - गाव, पत्ता / लोकेशन (Google Maps पिन किंवा गाव+वाडी निवड)

2. शेळी/बोकड एंट्री (Livestock Entry)
   - प्रकार: बोकड / शेळी / पिल्लू (करडू)
   - जात (Breed) - उस्मानाबादी, संगमनेरी, बीटल इ.
   - वय (महिने/वर्ष), वजन (अंदाजे किलो)
   - लिंग (नर/मादी)
   - जन्मतारीख (पिल्लू असल्यास - आपोआप वय कॅलकुलेट होईल)
   - फोटो अपलोड (२-३ फोटो)
   - अपेक्षित किंमत (ऐच्छिक - किंवा "बोलणी करून" ऑप्शन)
   - सध्या विक्रीसाठी आहे की नाही (टॉगल: Available / Not Available)
   - आरोग्य स्थिती / लसीकरण झालं आहे का (ऐच्छिक)

3. खातिक/खरेदीदार साईड
   - लॉगिन शिवायही लिस्टिंग बघता येईल (Public View) - कुणीही थेट वेबसाईट
     उघडून लिस्टिंग बघू शकेल, संपर्कासाठीच फक्त लॉगिन गरजेचं असेल
   - "सध्याचं लोकेशन वापरा" बटण - ब्राउझरच्या Geolocation ने खातिकचं current
     location घेऊन, सगळ्या एंट्री त्याच्या जवळपासून-ते-लांब (नजीकचं आधी)
     अशा क्रमाने दाखवायच्या (अंतर किमी मध्ये दाखवणे). गाव निवडून मॅन्युअल
     सर्च करायचा ऑप्शनही असेलच, पण "जवळचं आधी" हे डिफॉल्ट/मुख्य सर्च असेल
   - फिल्टर: प्रकार, वय, वजन, किंमत रेंज, जात
   - शेतकऱ्याचा नंबर दिसणे + "Call Now" बटण (फोनच्या डायलरला डायरेक्ट कनेक्ट
     - वेबसाईटमधून कुठलाही SMS/कॉल स्वतः पाठवला जाणार नाही, फक्त नंबर दाखवायचा)
   - शेतकऱ्याच्या घराचं लोकेशन (Google Maps लिंक - डायरेक्ट दिशादर्शन)

4. माझी एंट्री मॅनेज करणे (Farmer Dashboard)
   - स्वतःच्या सगळ्या एंट्री बघणे, एडिट/डिलीट करणे
   - "विकलं गेलं" मार्क करणे (झाल्यावर लिस्टिंगमधून आपोआप बाहेर)

5. सर्च आणि लिस्टिंग पेज
   - गावनिहाय / परिसरनिहाय यादी
   - कार्ड व्यू: फोटो + मूळ माहिती + अंतर (किती किमी दूर)

6. अॅडमिन पॅनेल
   - सगळे युजर्स, गावं, एंट्री बघणे
   - फेक/स्पॅम एंट्री काढणे
   - गावांची यादी मॅनेज करणे
   - "शेतकऱ्याच्या वतीने एंट्री टाका" - फोन कॉलवरून माहिती घेऊन अॅडमिन
     स्वतः एंट्री भरू शकेल (ज्या शेतकऱ्याला रजिस्ट्रेशन जमत नाही त्यासाठी)


पेजेस (Website Pages)
----------------------
- Home (गाव निवडा + जवळपासचे लिस्टिंग)
- Login / Register (मोबाईल नंबर + पासवर्ड)
- शेळी एंट्री टाका (फॉर्म - फोटो सहित)
- माझ्या एंट्री (Dashboard - फार्मरसाठी)
- सर्व लिस्टिंग / सर्च पेज (फिल्टरसह)
- एका एंट्रीचा डीटेल पेज (फोटो, माहिती, संपर्क बटण, मॅप)
- अॅडमिन पॅनेल


टेक स्टॅक (फायनल निवड)
-----------------------------------------------
- Next.js (App Router) + React + TypeScript - फ्रंटएंड आणि बॅकएंड दोन्ही
  एकाच Next.js अॅपमध्ये. वेगळा Fastify/Express सर्व्हर लागणार नाही -
  Next.js च्या API Routes (app/api/...) मधूनच backend logic चालेल.
  pulse प्रोजेक्टसारखं वेगळं monorepo/pnpm-workspace सेटअप इथे लागत नाही,
  कारण हा एक छोटा सिंगल-अॅप प्रोजेक्ट आहे.
- Styling: Tailwind CSS (मोबाईल-फर्स्ट - गावातले बहुतेक युजर्स मोबाईलवरूनच
  वापरतील म्हणून रिस्पॉन्सिव्ह डिझाईन महत्त्वाचं)
- Database: MySQL + Prisma ORM (TypeScript सेफ, MySQL सपोर्ट चांगला आहे)
- फोटो स्टोरेज: सुरुवातीला सर्व्हरवर लोकल फोल्डर (public/uploads), नंतर
  गरज पडली तर S3/Cloud स्टोरेजकडे शिफ्ट करता येईल
- ऑथ: साधं मोबाईल नंबर + पासवर्ड लॉगिन - OTP/SMS नाही (कुठलीही SMS API
  इंटिग्रेशन लागणार नाही, सगळं डेटाबेसमध्येच चेक होईल)
- मराठी भाषा सपोर्ट गरजेचं - संपूर्ण UI मराठीत ठेवणं (कारण युजर्स गावातले
  शेतकरी आणि खातिक आहेत, इंग्रजी UI त्यांना अवघड जाईल)

काय वापरणार नाही (स्पष्टपणे वगळलेलं)
-----------------------------------------------
प्रोजेक्ट अगदी साधा आणि शेतकरी/खातिकला सहज वापरता यावा म्हणून खालच्या
कुठल्याही गोष्टी या प्रोजेक्टमध्ये नसतील:
- कुठलाही Email पाठवणे/घेणे (no email sending/receiving)
- कुठलाही SMS / OTP पाठवणे (no SMS API, no OTP verification)
- कुठलंही Payment / पैसे ट्रान्झॅक्शन (no payment gateway, Stripe इ. नाही)
- WhatsApp/Push नोटिफिकेशन सारखं काहीही ऑटोमेटेड मेसेजिंग
संपर्क फक्त "Call Now" बटणाने होईल - जो फोनच्या नॉर्मल डायलरला उघडेल,
वेबसाईट स्वतः कुठलाही मेसेज/कॉल पाठवणार नाही.

साधारण फोल्डर स्ट्रक्चर:
  shelipalan/
    app/
      page.tsx              (Home - listing + गाव फिल्टर)
      login/
      entry/                (नवीन शेळी/पिल्लू एंट्री फॉर्म)
      dashboard/             (माझ्या एंट्री - फार्मर)
      listing/[id]/          (एका एंट्रीचा डीटेल पेज)
      admin/
      api/
        auth/
        listings/
        villages/
    prisma/
      schema.prisma          (User, Listing, Village मॉडेल्स)
    components/
    public/uploads/          (फोटो स्टोरेज)


पुढच्या टप्प्यातील आयडिया (Future Scope - सगळं अजूनही no-email/SMS/payment)
------------------------------------------------------------------------------
- शेळीचा संपूर्ण इतिहास ट्रॅक करणे (जन्म ते विक्री - एक प्रकारची "horoscope"
  सारखी हिस्ट्री: कोणत्या आईपासून जन्म, लसीकरण रेकॉर्ड, वजन वाढीचा ग्राफ)
- खातिकांचं रेटिंग/रिव्ह्यू सिस्टीम (शेतकऱ्यांना विश्वासू खातिक ओळखता यावा)
- बाजारभाव माहिती (आजचा बोकड/शेळीचा सरासरी भाव - गावनिहाय)
- एकाच गावातल्या सगळ्या शेळीपालकांचा ग्रुप / कम्युनिटी फीचर


UX - खास लक्षात ठेवायच्या गोष्टी (Must-haves)
--------------------------------------------------
0. टार्गेट युजर लक्षात ठेवा - हे "नॉर्मल" साधे शेतकरी आहेत, टेक्नो-सॅव्ही
   नाहीत. कुठलंही फीचर बनवण्याआधी हा प्रश्न विचारायचा: "हे एखाद्या साध्या
   शेतकऱ्याला सहज समजेल आणि वापरता येईल का?"
   - कुठलाही इंग्रजी टेक्निकल शब्द/जार्गन टाळायचा, साधी मराठी भाषा
   - आयकॉन्स + मोठी बटणं वापरायची, कमी टेक्स्ट
   - जास्त स्टेप्स/क्लिक्स लागणारे फ्लो टाळायचे

1. शेतकऱ्यासाठी (Farmer) - एंट्री टाकणं अगदी सोपं आणि पटकन व्हायला हवं:
   - नवीन पिल्लू/शेळी जन्मली की २-३ मिनिटात एंट्री टाकता यावी (कमी फॉर्म
     फील्ड्स, मोठी बटणं, फोटो काढून थेट अपलोड)
   - "माझ्या एंट्री" पेज उघडल्यावर लगेच स्वतःचा स्टॉक स्पष्ट दिसावा
     (कार्ड्स/फोटो सहित - टेक्स्ट-हेवी लिस्ट नको)

2. खातिकसाठी (Buyer) - लोकेशन बेस्ड सर्च हे या प्रोजेक्टचं सगळ्यात
   महत्त्वाचं फीचर आहे:
   - वेबसाईटवर आल्यावर सध्याचं (current) लोकेशन वापरून आपोआप जवळपासच्या
     एंट्री दिसाव्यात - "जवळपासच्या आधी, लांबच्या नंतर" असं सॉर्टिंग
   - प्रत्येक एंट्रीवर अंतर (उदा. "२.५ किमी दूर") स्पष्ट दिसावं
   - मॅन्युअल गाव/परिसर सर्चचा ऑप्शनही ठेवायचा, पण current-location सर्च
     हाच डिफॉल्ट आणि सगळ्यात ठळक असेल

3. ज्यांना रजिस्ट्रेशन/वेबसाईट वापरता येत नाही त्यांच्यासाठी - फोन कॉलवरून
   एंट्री (Assisted Entry by Admin):
   - काही शेतकऱ्यांना स्वतः रजिस्ट्रेशन/एंट्री करता येणार नाही (मोबाईल
     वापरता येत नाही, वेबसाईट समजत नाही इ.) - असे शेतकरी डायरेक्ट फोन
     करून (तुम्हाला/अॅडमिनला) माहिती सांगतील
   - अॅडमिन पॅनेलमध्ये एक खास ऑप्शन असेल - "शेतकऱ्याच्या नावाने एंट्री टाका"
     - अॅडमिन फोनवर ऐकलेली माहिती (नाव, मोबाईल नंबर, गाव, शेळीची डिटेल्स)
       स्वतः एंट्री फॉर्ममध्ये भरून सेव्ह करेल
     - अशा एंट्रीसाठी वेगळा रजिस्ट्रेशन/पासवर्ड लागणार नाही - अॅडमिनच
       शेतकऱ्याच्या वतीने ती एंट्री मॅनेज करू शकेल (नंतर हवं तर शेतकरी
       स्वतःचा मोबाईल नंबर वापरून लॉगिन करूनही ती एंट्री बघू शकेल)


पहिल्या व्हर्जनसाठी (MVP - किमान आवश्यक गोष्टी)
--------------------------------------------------
सुरुवातीला फक्त एवढं असलं तरी पुरेल:
1. फार्मर लॉगिन/रजिस्टर
2. शेळी/पिल्लू एंट्री टाकणे (फोटो + बेसिक माहिती + संपर्क नंबर) - सोपा फॉर्म
3. सगळ्यांना दिसणारी पब्लिक लिस्टिंग पेज - current-location वापरून
   जवळपासच्या एंट्री आधी दाखवणे (+ गावनिहाय मॅन्युअल फिल्टरही)
4. डीटेल पेजवर "Call Now" बटण

हे झालं की मग टप्प्याटप्प्याने वरचे फीचर्स अॅड करता येतील.
